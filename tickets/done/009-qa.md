Both warnings are `KEY_FIREFOX_UNSUPPORTED_BY_MIN_VERSION` / `KEY_FIREFOX_ANDROID_UNSUPPORTED_BY_MIN_VERSION` — the manifest declares `data_collection_permissions` (introduced in Firefox 140/142) while `strict_min_version` is `109.0`. This key was added to the manifest in a prior ticket and was already present before ticket 009 touched any files. The `content.js`, `content.css`, and `popup.js` changes in this ticket do not affect the manifest and did not introduce these warnings.

> **Recommended follow-up:** Either bump `strict_min_version` to `140.0` or remove `data_collection_permissions` from the manifest if it is not required. Neither is a blocker for this ticket.

- `content.js:86-90` — Storage gate: `browser.storage.local.get('enabled').then(({ enabled = true }) => { if (!enabled) return; ... })`. Correct default-true semantics; exits before TreeWalker or DOM mutation when disabled. ✓
- `content.css:12` — `body.pinyin-hidden rt { display: none; }` appended after existing rules. Selector specificity is sufficient; no conflicts with existing rules. ✓
