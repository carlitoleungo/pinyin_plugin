#### Notes on test setup

- Content script logs appear in the **page DevTools console** (F12 → Console), not the Browser Console (`Cmd+Shift+J`). The ticket's Notes for QA correctly says F12 → Console.
- Local `file://` URLs are not matched by `*://*/*` — Firefox blocks extension access and shows a permission warning. This is expected browser behaviour, not a bug. Do not use local HTML files for content script testing without adding `"file://*/*"` to matches (out of scope for this ticket).
