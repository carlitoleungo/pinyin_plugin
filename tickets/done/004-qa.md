**Step 3 — AC1: Static code review of requestIdleCallback implementation**

`content_scripts/content.js` lines 58–79:

```js
function processChunk(nodes, index) {
  const CHUNK_SIZE = 100;
  const end = Math.min(index + CHUNK_SIZE, nodes.length);
  for (let i = index; i < end; i++) {
    if (!nodes[i].parentNode) continue;   // ← correct: continue, not break
    annotateTextNode(nodes[i]);
  }
  if (end < nodes.length) {
    scheduleChunk(nodes, end);             // ← recurses with correct next index
  }
}

function scheduleChunk(nodes, index) {
  if (typeof requestIdleCallback !== 'undefined') {
    requestIdleCallback(() => processChunk(nodes, index));   // ← primary path
  } else {
    setTimeout(() => processChunk(nodes, index), 0);         // ← fallback
  }
}

const chineseNodes = findChineseTextNodes();
scheduleChunk(chineseNodes, 0);   // ← defers even first chunk; no sync annotation
```

Checklist:
- `requestIdleCallback` guard present ✓
- `setTimeout` fallback present ✓
- CHUNK_SIZE = 100 ✓
- Stale node guard uses `continue` (not `break`) ✓
- Recursion passes `end` (not `index + CHUNK_SIZE`) — correct since `end = Math.min(...)` ✓
- Entry point defers first chunk (no immediate sync call) ✓
- `findChineseTextNodes` runs synchronously (collection only — within spec) ✓

AC1 PASS.
