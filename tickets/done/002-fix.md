**Root cause of AC3 failure (for the record):**
The original AC3 said to run `pinyinPro.pinyin('汉')` in the F12 page console. Firefox content scripts execute in an isolated sandbox — `var pinyinPro` is scoped to that sandbox, not the page's `window`. The page console evaluates in the page context and cannot see content script variables. This is correct Firefox behavior, not a bug in the bundle. `content.js` shares the same sandbox as `vendor/pinyin-pro.js` and has full access to `pinyinPro`.
